# AIFI - AI Finance Index

## Product Specification v1.0

---

## Overview

AIFI is a curated directory of companies building at the intersection of AI and Finance. The site helps investors and builders discover projects, understand the competitive landscape, and navigate the industry through two primary views: a searchable **Directory** and visual **Market Maps**.

**Goals:**
- Be the definitive source of truth for AI + Finance companies
- Simple to browse, simple to maintain
- Static-first architecture for reliability and speed

---

## Core Features

### 1. Project Directory

A searchable, filterable list of all indexed companies.

**Project Card displays:**
- Company name and logo
- One-line description
- Primary segment and layer badges
- Funding stage
- Headquarters location

**Filtering:**
- By market segment (10 options)
- By tech stack layer (9 options)
- By funding stage
- By geography

**Search:**
- Full-text search across name, description, and tags

**Sorting:**
- Alphabetical
- Most recently added
- By funding amount (if known)

### 2. Project Profile

Individual page for each company with expanded details.

**Contents:**
- Company name, logo, tagline
- Full description
- Website and social links
- Market segment(s)
- Tech stack layer(s)
- Funding history (stage, amount, date)
- Key team members (optional)
- Similar companies (auto-generated by segment + layer)

### 3. Market Maps

Visual representations of the competitive landscape.

**Segment Maps:**
- Show all companies within a single market segment
- Organized by tech stack layer (rows)
- Companies displayed as logo tiles

**Layer Maps:**
- Show all companies at a single tech stack layer
- Organized by market segment (columns)

**Features:**
- Click any company to go to their profile
- Download as image (for presentations/reports)

### 4. Browse Pages

**Segments Index:**
- List all 10 market segments
- Show company count per segment
- Brief description of each

**Layers Index:**
- List all 9 tech stack layers
- Show company count per layer
- Brief description of each

## Content Model

### Project
```
- name (required)
- slug (required, auto-generated)
- logo_url
- tagline (max 140 chars)
- description (markdown)
- website_url
- twitter_url
- linkedin_url
- primary_segment (required)
- secondary_segments[]
- primary_layer (required)
- secondary_layers[]
- stage (pre_seed | seed | series_a | series_b | series_c_plus | growth | public | acquired | bootstrapped)
- total_funding_usd
- founded_year
- headquarters_country (ISO code)
- headquarters_city
- team_members[] (name, role, linkedin_url)
```

### Segment
```
- name
- slug
- description
- icon
- color
```

### Layer
```
- name
- slug
- description
- position (1-9, bottom to top)
- icon
- color
```

## Technical Approach

**Static Site Generation**
- Build once, deploy everywhere
- No backend servers to maintain
- Data stored in JSON/Markdown files

**Recommended Stack:**
- Next.js (static export)
- Tailwind CSS
- Content in `/data` folder as JSON
- Deployed to Vercel or Netlify

**Data Management:**
- Edit JSON files directly or via simple admin script
- Rebuild and deploy on content changes
- No database required

**Search:**
- Client-side search with Fuse.js
- Fast enough for 500-1000 projects
- No external search service needed

---

## Design Direction

**Aesthetic:**
- Clean, professional, data-forward
- Dark mode by default (financial/tech feel)
- Dense but readable information display

**Key Principles:**
- Every page should load instantly
- Filtering should feel immediate
- Mobile-friendly but desktop-optimized

---

## Maintenance

**Adding a new company:**
1. Add entry to `data/projects.json`
2. Add logo to `public/logos/`
3. Rebuild and deploy

**Updating a company:**
1. Edit entry in `data/projects.json`
2. Rebuild and deploy

**No ongoing costs except:**
- Domain name
- Hosting (free tier sufficient)

---

## MVP Scope

**Phase 1 (Launch):**
- 100-200 curated projects
- Directory with search and filters
- Individual project profiles
- Segment and layer browse pages
- One market map (full grid view)

**Phase 2 (If needed):**
- Individual segment maps
- Individual layer maps
- Newsletter signup
- Project submission form
